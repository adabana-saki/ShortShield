# ShortShieldへのコントリビューション

ShortShieldへのコントリビューションに興味を持っていただきありがとうございます！このドキュメントはコントリビューターのためのガイドラインと情報を提供します。

[English](CONTRIBUTING.md)

## 行動規範

行動規範をお読みいただき、従ってください。すべてのコントリビューターは敬意を持ち、建設的であることを期待しています。

## コントリビューションの方法

### バグの報告

1. [Issues](https://github.com/adalab/shortshield/issues)でバグがすでに報告されていないか確認
2. 報告されていない場合、バグレポートテンプレートを使用して新しいIssueを作成
3. 以下を含めてください：
   - ブラウザとバージョン
   - 拡張機能のバージョン
   - 再現手順
   - 期待される動作と実際の動作
   - 該当する場合はスクリーンショット

### 機能の提案

1. 既存の[Issues](https://github.com/adalab/shortshield/issues)で類似の提案がないか確認
2. 機能リクエストテンプレートを使用して新しいIssueを作成
3. 以下を記述：
   - 解決しようとしている問題
   - 提案するソリューション
   - 検討した代替ソリューション

### コードのコントリビューション

#### セットアップ

1. リポジトリをフォーク
2. フォークをクローン：
   ```bash
   git clone https://github.com/YOUR_USERNAME/shortshield.git
   cd shortshield
   ```
3. 依存関係をインストール：
   ```bash
   pnpm install
   ```
4. ブランチを作成：
   ```bash
   git checkout -b feature/your-feature-name
   ```

#### 開発ワークフロー

1. 変更を加える
2. コード品質を確認：
   ```bash
   pnpm lint
   pnpm typecheck
   pnpm test:unit
   ```
3. ブラウザで手動テスト
4. 変更をコミット（以下のコミットガイドラインを参照）
5. フォークにプッシュ
6. プルリクエストを開く

#### コミットガイドライン

Conventional Commitsを使用します。フォーマット：

```
type(scope): description

[オプションの本文]

[オプションのフッター]
```

タイプ：
- `feat`: 新機能
- `fix`: バグ修正
- `docs`: ドキュメントの変更
- `style`: コードスタイル（フォーマット、セミコロンなど）
- `refactor`: コードリファクタリング
- `test`: テストの追加または更新
- `chore`: メンテナンスタスク

例：
```
feat(content): TikTokブロック機能を追加
fix(popup): トグル状態が保持されない問題を修正
docs(readme): インストール手順を更新
```

#### プルリクエストガイドライン

1. プルリクエストテンプレートを使用
2. 関連するIssueを参照
3. UI変更の場合はスクリーンショットを含める
4. すべてのチェックがパスしていることを確認
5. メンテナにレビューをリクエスト

### コードスタイル

- TypeScript strictモード
- ESLintとPrettierでフォーマット
- コードベースの既存パターンに従う

### テスト

- 新機能にはテストを書く
- テストカバレッジを80%以上に維持
- 提出前にテストを実行：
  ```bash
  pnpm test:unit
  pnpm test:e2e
  ```

### ドキュメント

- 機能を追加する場合はREADMEを更新
- 公開APIにはJSDocコメントを追加
- 文字列を追加する場合は翻訳を更新

## セキュリティ

- 機密データをコミットしない
- すべてのユーザー入力を検証
- セキュアコーディングプラクティスに従う
- セキュリティ問題は非公開で報告（SECURITY.mdを参照）

## 質問がありますか？

- 一般的な質問はDiscussionを開く
- コミュニティチャットに参加（近日公開）
- 既存のIssueとDiscussionを確認

## ライセンス

コントリビューションすることにより、あなたの貢献がMITライセンスの下でライセンスされることに同意するものとします。
